# How to Cite

PDSL documentation should be cited using:

1) the tagged release (recommended), and  
2) the repository citation file (`CITATION.cff`).

## Recommended citation practice

- Cite the **tagged release** (e.g., `v0.1.1-review-ready`) for stable reference.
- If you must cite the live site, include the access date.

## What not to cite PDSL as
PDSL is not a forecasting system and does not perform causal inference. Avoid framing PDSL as:
- predictive modeling,
- automated decision-making,
- treatment-effect estimation.

## Repository citation file
Use the repository’s `CITATION.cff` for standardized metadata.
