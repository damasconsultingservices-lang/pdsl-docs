# How to Cite

PDSL documentation should be cited using:

1) the tagged release (recommended), and  
2) the corresponding Zenodo DOI, and  
3) the repository citation file (`CITATION.cff`) for standardized metadata.

---

## Recommended citation practice

- Cite the **tagged release** (e.g., `v0.1.2-review-ready`) for stable reference.
- Use the **Zenodo DOI (citable archive)** for version-anchored citation:

  **DOI (Zenodo, citable documentation release):**  
  https://doi.org/10.5281/zenodo.18732275

- Avoid citing untagged commits or the `main` branch.
- If referencing the live documentation site, include an access date.

---

## Example citation (documentation release)

Damas, F. (2026). *Public Diplomacy Simulation Lab (PDSL): Public Documentation Release v0.1.2-review-ready*. Zenodo. https://doi.org/10.5281/zenodo.18732275

---

## What not to cite PDSL as

PDSL is not a forecasting system and does not perform causal inference. Avoid framing PDSL as:

- predictive modeling,
- automated decision-making,
- treatment-effect estimation,
- empirical measurement of real-world outcomes.

PDSL provides regime-based interpretive constructs for structured analytical reasoning.

---

## Repository citation metadata

For machine-readable citation data, refer to the repository’s `CITATION.cff`, which mirrors the tagged release and Zenodo DOI.
