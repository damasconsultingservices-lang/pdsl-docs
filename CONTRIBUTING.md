# Contributing to PDSL Docs

Thanks for improving the public documentation for the Public Diplomacy Simulation Lab (PDSL).

## Scope
This repository is documentation-first and review-facing. Contributions should:
- improve clarity, auditability, and internal consistency
- reduce ambiguity in definitions and non-claims
- add references, examples, or testable documentation improvements

## Not accepted
- claims that PDSL “predicts” real-world outcomes
- operational influence tooling, targeting guidance, or manipulation playbooks
- content that encourages misuse of the framework

## How to contribute
1) Create a branch
2) Make focused commits
3) Open a Pull Request (PR) that includes:
   - what changed
   - why it matters (reviewer lens)
   - any risks or assumptions

## Style rules
- Prefer short paragraphs
- Define acronyms at first use
- Keep “non-claims” visible
- Use consistent naming for APR/IAS/PST and core concepts
