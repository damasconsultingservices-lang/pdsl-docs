# Security Policy

## Supported versions
This repository publishes static documentation. Security issues most commonly relate to:
- dependency vulnerabilities in the documentation build chain
- unsafe links or injected scripts (should not exist)

## Reporting a vulnerability
If you believe you found a security issue:
- Do not open a public issue with exploit details.
- Contact the repository owner/maintainer with:
  - a clear description
  - steps to reproduce
  - impact assessment
  - any recommended fix

## Response expectations
We aim to acknowledge reports and assess severity quickly, then patch dependencies or content as appropriate.
